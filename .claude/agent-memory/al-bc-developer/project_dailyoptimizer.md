**Embedded resource files (added 2026-07-08):** Bundling a static file (e.g. CSV lookup data) into the `.app` requires declaring `"resourceFolders": ["src/data"]` in `app.json` â€” this is a BC 2024 wave 2 / update 25.2+ feature and is NOT automatic; plain loose files anywhere in the project (project root, an undeclared folder, etc.) are silently dropped from the package even though `al_build` reports success. Verified by actually unzipping the built `.app` (it's a ZIP with a ~40-byte header before the `PK\x03\x04` signature) and checking for a `PackagedResources.json` entry plus `resources/<file>` entries. `NavApp.GetResourceAsText(ResourceName, TextEncoding::UTF8)` (and `GetResourceAsStream`/`GetResourceAsJson`) take `ResourceName` **relative to the declared folder**, not the project-relative path â€” e.g. `src/data/FirstNames.csv` on disk is read back as just `'FirstNames.csv'`. `codeunit_50602_CreateDemoData.al`'s `GetNextName()` uses these CSVs (635 first names x 1000 last names = 635,000 combos) with a collision-free permutation â€” `PermutedIndex := (Counter * 392453) mod 635000` computed in `BigInteger` (the raw `Counter * Multiplier` product overflows 32-bit `Integer` well before the ~34,725 resources created per demo-data run) â€” 392453 was chosen because it's coprime with 635000 (`= 2^3 * 5^4 * 127`), making the mapping a bijection over `[0, 635000)` so every drawn name pair is unique as long as draws stay under 635,000.
